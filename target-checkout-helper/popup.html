Quick tips for the fastest checkout
  1. Stay logged in on Target in a dedicated window.
  2. On the Shipping & payment tab, turn on Use saved payment & address when your card and address are saved on Target.
  3. Enable the extension, open a product page (/p/…), then let it add to cart and continue to checkout.
  4. For drops, set Expected drop time so polling tightens near go-time (same idea as only going “live” right before go-time elsewhere).
  5. Use one Target tab with this extension per profile so sessions and retries do not fight each other.
  6. Clear your cart on Target before a checkout run so old items do not block or confuse the flow.
  7. On PC (especially on Ethernet), some buyers restart the connection and clear stale Target cookies for a fresher session—you can keep browser autofill. Results vary; never share passwords.

    1 — Load the extension

    1. Go to chrome://extensions
    2. Enable Developer mode (top-right toggle)
    3. Click Load unpacked → select the target-checkout-helper folder
    4. Pin it to your toolbar via the puzzle icon

    2 — One-time setup

    1. Sign in to Target in Chrome and make sure your shipping address and credit card are saved on your account — the bot uses those directly
    2. Open this popup → turn Enable ON
    3. Turn Use saved payment & address ON — this is the fast path, skips all form filling
    4. Turn Auto place order ON if you want fully automatic checkout
    5. Set Retry max → 0 (unlimited) and Retry delay → 1s

    3 — Cookie harvesting

    Keeps your session fresh so Target doesn't reject checkout mid-run.

    1. In the Monitor & options tab find Cookie harvest
    2. Turn Harvesting on → ON
    3. Set Snapshots per page load → 2
    4. Turn Don't stop harvesting → ON
    5. Turn Apply before checkout → ON
    6. Leave expiration at 3 min, order at LIFO
    7. Browse Target normally while you wait — each visit auto-snapshots. Watch Snapshots ready: N stay above 0

    4 — Set up the monitor

    1. In Monitor & options paste each product URL and hit +
    2. Set qty to 5 per product
    3. If you know the drop time set it in Expected drop / restock — polling jumps to 250ms in the 10-minute window
    4. Hit Start monitoring — the extension opens one background tab per product automatically
    5. Keep Chrome open with at least one Target tab visible

    5 — While you wait

    1. Don't close Chrome — minimized is fine
    2. Every 10–15 min reload a Target tab to keep cookies fresh
    3. Watch the popup status line — it shows the poll is running
    4. Don't flip extension settings in the last minute before drop time
    5. Make sure your Target cart is empty before the drop

    6 — What happens on restock

    1. Background poll detects stock via RedSky API
    2. Navigates the background tab to the product page
    3. Clicks ATC using the debugger (human mouse movement)
    4. Drives through checkout using your saved address & card
    5. Clicks Place Order automatically
    6. Toast appears bottom-right tracking each step

    Watch for

    • "Sign in if you see a prompt" — Target hit an auth gate. Sign in manually in that tab, bot continues automatically
    • "Snapshots ready: 0" — harvest pool empty, browse a Target page to refill
    • Chrome banner "controlled by automated software" — normal, means debugger clicks are working
    • Cart not empty — clear your Target cart before the drop or old items may block checkout
